# AI-PREFERENCES

适用场景：

- 发布 GitHub Release
- 整理文档或规则
- 需要判断输出风格、协作方式时

## 长期偏好

- 默认直接动手做，少把零碎步骤再丢给用户
- 发布 GitHub Release 时，默认写给用户看的更新说明，不写内部技术细节
- 文档尽量短：核心规则留在 `AGENTS.md`，细节放到按需查看的 runbook
- `项目日志.md` 只记关键变化和仍影响后续判断的内容，早期不重要历史可省略
- AI 默认不要通读全部文档，应先看路由，再按任务加载必要文档

## 维护规则

- 这里只保留稳定、重复出现、长期有效的偏好
- 新偏好应优先合并或替换旧表述，不堆相似条目
- 如果某条偏好已经变成项目公共规则，应迁移到 `AGENTS.md`
