# 版本更新日志

## v1.0.10 (2026-03-25)
- 增强列表页视频封面查看体验：灯箱大图模式现在支持左右导航按钮切换同一视频的不同封面。
- 灯箱大图模式新增鼠标滚轮切换图片功能。
- 灯箱大图模式新增键盘左右方向键和 Esc 键操作支持。

## v1.0.9 (2026-03-25)
- 播放预热架构升级：background 新增 m3u8 预热与会话缓存（`GET_PREFETCH_M3U8`），播放器可直接命中预热转码列表。
- 修复“只剩无损、没有115原画/转码档位”问题：双通道现在同步消费 `Ultra + m3u8` 两类缓存并回填完整画质菜单。
- 进一步收敛播放链路到“预热缓存命中优先 + 实时请求兜底”，减少刷新后首播冷启动抖动。

## v1.0.8 (2026-03-25)
- 播放启动链路改为真正双通道竞速：`m3u8` 与 `Ultra` 并发请求，谁先可用先播，显著降低首播等待时间。
- 修复画质选项只剩“无损”的问题：完成竞速后统一汇总 `Ultra + m3u8` 画质列表，恢复“无损/115原画/其它转码”可选。
- 为 `m3u8` 增加轻量重试，提升首播阶段对网络抖动的容错。

## v1.0.7 (2026-03-25)
- 解决“刷新后首播慢、二次秒播”核心缓存断层：预热无损地址缓存新增 `chrome.storage.session` 持久层，避免仅存内存导致页面刷新后冷启动。
- 首页新增首屏视频源自动预热（最多 3 个可见视频），降低首次点击后的首播等待时间。
- 保持点击即开标签与播放器进度条缩略图预览能力。

## v1.0.6 (2026-03-25)
- 修复“点击后长时间无反馈”交互问题：点击视频时先立即打开 `about:blank` 新标签，再无缝跳转到扩展播放器页，保证用户点击后立刻有可见反馈。
- 播放点击事件从等待列表就绪改为主文档先绑定、目标文档就绪后补绑定，避免首次点击落在未绑定窗口期。
- 保留后台 `OPEN_TAB` 作为兜底路径，确保在极端弹窗策略下仍可打开播放器页。

## v1.0.5 (2026-03-25)
- 修复首次点击视频偶发长时间无响应：新增后台 Service Worker 预热 `PING`，并在点击时并行预热+开页，降低冷启动首击延迟。
- 恢复后台 `OPEN_TAB` 单通道开页，避免受浏览器弹窗策略影响导致首击不稳定。
- 统一 content/player 侧 runtime 消息安全封装，遇到扩展热更新导致的 context invalidated 时降噪处理。
- 播放器新增进度条悬停缩略图预览（使用已有封面数据显示预览图和时间）。

## v1.0.4 (2026-03-25)
- 彻底移除“当前页回退打开播放器”行为，`window.open` 失败时仅提示，不再改写原标签页地址，从根源杜绝“原页也变播放页”。
- 增加全局捕获阶段视频点击拦截，在最早事件阶段阻断 115 原生视频打开链路，确保仅由扩展播放器流程接管。
- 显式跳过 `chrome-extension://.../src/player/index.html` 页面注入，避免播放器页被 content script 误处理。

## v1.0.3 (2026-03-25)
- 从根本重构点击播放链路：改为文档级单入口捕获拦截（`mousedown + click + dblclick`），统一在一个入口打开播放器，彻底避免节点级多处绑定引发的并发跳转。
- 移除逐文件项绑定播放事件逻辑，杜绝列表重渲染后重复绑定导致的双开问题。
- 保留视频源预热，但改为文档级触发，减少事件分散造成的行为不一致。

## v1.0.2 (2026-03-25)
- 从底层收敛播放打开链路为单通道：仅使用 `window.open`，不再与后台开页并行，彻底避免双标签竞态。
- 移除视频项双击打开逻辑，仅保留单击入口，消除双击/单击叠加触发风险。
- 给每个文件项增加一次性点击绑定标记，阻止重复绑定导致的多次触发。
- 在文件名节点捕获阶段拦截 `mousedown`，防止 115 原生逻辑抢占并触发并发开页。
- 增加全局初始化幂等锁，杜绝同页面重复初始化造成的事件重复注册。

## v1.0.1 (2026-03-25)
- 修复刷新后首击视频偶发重复开页，增加同视频短时防重复打开。
- 增加仅顶层页面注入保护，避免子 frame 重复绑定事件。
- 调整文件列表等待日志输出频率，减少控制台刷屏。
- 扩展名称更新为“115Master移植版”。
- 新增扩展图标（16/32/48/128）。
