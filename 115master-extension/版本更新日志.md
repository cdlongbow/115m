# 版本更新日志

## v1.0.4 (2026-03-25)
- 彻底移除“当前页回退打开播放器”行为，`window.open` 失败时仅提示，不再改写原标签页地址，从根源杜绝“原页也变播放页”。
- 增加全局捕获阶段视频点击拦截，在最早事件阶段阻断 115 原生视频打开链路，确保仅由扩展播放器流程接管。
- 显式跳过 `chrome-extension://.../src/player/index.html` 页面注入，避免播放器页被 content script 误处理。

## v1.0.3 (2026-03-25)
- 从根本重构点击播放链路：改为文档级单入口捕获拦截（`mousedown + click + dblclick`），统一在一个入口打开播放器，彻底避免节点级多处绑定引发的并发跳转。
- 移除逐文件项绑定播放事件逻辑，杜绝列表重渲染后重复绑定导致的双开问题。
- 保留视频源预热，但改为文档级触发，减少事件分散造成的行为不一致。

## v1.0.2 (2026-03-25)
- 从底层收敛播放打开链路为单通道：仅使用 `window.open`，不再与后台开页并行，彻底避免双标签竞态。
- 移除视频项双击打开逻辑，仅保留单击入口，消除双击/单击叠加触发风险。
- 给每个文件项增加一次性点击绑定标记，阻止重复绑定导致的多次触发。
- 在文件名节点捕获阶段拦截 `mousedown`，防止 115 原生逻辑抢占并触发并发开页。
- 增加全局初始化幂等锁，杜绝同页面重复初始化造成的事件重复注册。

## v1.0.1 (2026-03-25)
- 修复刷新后首击视频偶发重复开页，增加同视频短时防重复打开。
- 增加仅顶层页面注入保护，避免子 frame 重复绑定事件。
- 调整文件列表等待日志输出频率，减少控制台刷屏。
- 扩展名称更新为“115Master移植版”。
- 新增扩展图标（16/32/48/128）。
