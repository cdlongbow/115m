# 115m 开发约定

## 基础原则

- 不做补丁式堆叠，改为模块化扩展。
- 新能力先定义边界，再落代码。
- 一次改动尽量只改一个层级职责。

## 分层边界

- `src/content/`：115 页面侧交互与列表增强。
- `src/player/`：播放器页逻辑与播放体验。
- `src/background/`：扩展后台消息路由、缓存、下载、历史。
- `src/shared/`：跨层共享协议与类型。

## 消息协议规则

- 所有新增消息必须先在 `src/shared/messages.ts` 定义。
- content/player/background 只引用共享协议，不重复本地定义。

## 提交流程

1. 本地修改代码
2. 执行 `pnpm build`
3. 按 `开发验收清单.md` 做手工冒烟
4. 再提交代码

## 命令规则

- 开发验证：`pnpm build`
- 发布打包：`pnpm zip`
- 不恢复 `pnpm dev` 作为主流程

## 仓库纯净规则

- 发布产物目录 `release/` 不入库。
- 新增目录前先考虑是否应被 `.gitignore` 排除。
- 不提交无关临时文件、测试截图、草稿脚本。
