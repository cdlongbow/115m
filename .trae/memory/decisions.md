# 技术决策记忆

## 用途

- 记录长期有效的技术决策、模块边界和维护约定
- 不记录工具迁移、历史流水账或已废弃入口

## 规则与记忆边界

- `.trae/rules/project-baseline.md` 是唯一始终生效规则入口，保持短规则
- 任务型规则按需保留，例如发布、提交信息等，不做 alwaysApply
- 需要记忆的项目事项统一写入 `.trae/memory/`，不使用 mem0 作为项目记忆来源
- `.trae/memory/*.md` 只记录会影响后续工作的项目级记忆，不放硬规则
- `docs/` 放项目说明、模块地图、runbook，不放强制执行规则
- 同一条规则不要在多个地方维护不同版本

## Chrome 扩展加载与安全边界

- 页面早期接管脚本 `public/video-page-early.js` 只负责同步写入最小播放器壳，不读取 Vite manifest，不动态 import 构建产物
- `src/content/video-page.ts` 不应运行时动态 import `../player/player`
- 涉及 content script、页面上下文、动态 import、Vite chunk 时，先确认执行环境和构建产物路径
- background 高权限消息需要 sender 来源校验；MAIN world 请求代理需要 URL 白名单

## 115 在线解压

- 在线解压复用 115 云端接口，不做本地下载解压再上传
- 单个解压支持需要密码时提示输入密码
- 批量解压默认串行执行，限制批量数量并避免并发；遇到需要密码的压缩包跳过并汇总提示，不在批量流程中逐个弹密码
- 解压完成后刷新列表优先复用后台刷新链路，页面内刷新只作兜底

## 播放器功能边界

- 缩略图功能必须与主播放链隔离，不能影响播放器初始化、播放源解析、播放列表主功能
- 播放列表是播放器右侧外置面板，不应覆盖视频或遮挡播放器右侧功能区
- 字幕控件在控制栏中应保持短标签或固定宽度，不直接展示过长原始字幕名，避免挤压倍速、播放模式等右侧控件
- 播放列表切换视频时，不能复用旧视频的 hover preview session、旧缩略图异步结果或旧尺寸状态
- 播放进度记忆以 115 原生 history 接口为准，本地 playHistory 代码保留但默认不参与读写
- 播放历史写入需要合并快速拖动/点击，只记录最后一次有效进度，避免高频写入
- 无损播放默认走保守兼容策略：稳定格式可默认原生，MKV 等不稳定格式只尊重用户手动选择，并在无声时自动回退到 115 原画

## UI 交互边界

- 文件夹封面收藏角标：已收藏常显，未收藏默认隐藏，鼠标悬浮或键盘聚焦时显示
- 文件夹封面收藏角标样式应与封面融合，采用嵌入式右上角折角，不做强分离的浮层按钮
