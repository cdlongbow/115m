# 项目核心规则

- 项目为 Chrome 扩展，统一使用 `pnpm`
- 默认直接在 `main` 做小改动、可快速验证的修复与迭代
- 修改代码前先看当前文件，再看相关代码，不默认通读全部文档
- 先理解再改；优先小改、稳改
- 只修改当前需求涉及的功能与边界，不自行扩展需求
- 未被当前需求明确要求、且已稳定的功能，禁止顺手改动、联动改样式、重构或行为调整
- 涉及共享模块或相邻稳定功能时，先检查模块、样式、状态和消息边界是否已隔离
- 如存在影响其他功能的风险，先完成隔离再继续功能改动
- 不把新逻辑继续堆进大文件，优先复用现有模块
- 发现本次改动产生了长期有效的规则、偏好、流程或模块边界时，主动同步更新对应文档或规则
- 新增或修改 `chrome.runtime` 消息时，同步更新 `src/shared/messages.ts`
- 改代码、功能、配置后执行 `pnpm build`
- 仅修改文档、说明或 `.trae/rules/*.md` 时，不默认执行 `pnpm build`，除非用户明确要求或改动影响构建/产物判断
- 改到逻辑、状态、消息、播放器核心链路时，同时执行 `pnpm test`
- 构建汇报使用中文，包含：结果 / 命令 / 关键报错 / 是否继续修复
- 不泄露密钥、账号、环境信息；不做未经确认的破坏性操作
