# Git 提交信息规则

- 提交信息采用“[类型] 具体问题或目标”的中文格式
- 类型示例：`[修复]`、`[功能]`、`[优化]`、`[重构]`、`[文档]`
- 摘要必须写清楚解决了什么问题或完成了什么目标，避免空泛描述
- 不写过长摘要，不堆砌变更细节；通常控制在一行，必要时再补一行说明
- 不使用英文提交前缀

## 推荐示例

- `[修复] 修复播放列表开启后进度预览图比例异常`
- `[修复] 修复移动弹窗按钮文字换行问题`
- `[优化] 播放历史写入合并快速拖动进度`
- `[功能] 接入 115 原生播放进度记忆`

## 避免示例

- `[功能] 优化播放器交互及预览功能`
- `[修复] 修复问题`
- `[优化] 调整代码`
