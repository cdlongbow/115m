# 版本更新日志

## v1.0.30 (2026-03-27)
- 修复播放器页面面包屑导航不显示的问题。
- 根因：之前从 115 页面 DOM（iframe 内）提取面包屑路径，但跨域 iframe 无法访问导致始终获取失败。
- 改为通过 115 文件列表 API（`getPlaylist`）返回的 `path` 字段获取目录路径，完全不依赖 DOM。
- 播放器页面加载后主动请求 API 获取面包屑，打开播放列表时也会同步更新。

## v1.0.29 (2026-03-27)
- 新增文件列表滚动位置记忆：切换目录后再返回，自动恢复到之前浏览的位置。
- 数据存储在 sessionStorage 中，浏览器标签关闭后自动清除。

## v1.0.28 (2026-03-27)
- 修复播放器 UI 可见时点击视频区域无法切换播放/暂停的问题。
- 根因：事件拦截逻辑将控制栏/底部栏整个容器视为交互控件，导致点击空白处被跳过。
- 现在只有按钮、进度条、设置面板等真正的交互元素才会阻止播放/暂停，其他区域统一响应点击切换。

## v1.0.26 (2026-03-27)
- 引入最小自动化测试层：新增 `Vitest`，为播放器核心纯逻辑模块补上单元测试。
- 覆盖画质选项构建、原画占位解析、播放状态同步、HLS 回退等关键逻辑，降低后续重构风险。
- 补测过程中顺手修复画质默认态判断：当选择 `115原画` 占位项时，不再同时把无损项也标记为默认选中。

## v1.0.25 (2026-03-27)
- 继续拆分播放器事件绑定层：将 `ready`、`loadedmetadata`、`canplay`、`playing`、`error`、进度保存与键盘绑定统一收口到独立模块。
- 清理播放器内残留的无效状态字段，进一步降低 `player.ts` 的维护复杂度。
- 重构后重新构建验证通过，无损优先、HLS 按需加载与画质切换行为保持不变。

## v1.0.24 (2026-03-27)
- 继续拆分播放器状态管理：将画质选项构建、当前播放源状态同步、HLS 回退状态切换抽到独立模块。
- `player.ts` 进一步聚焦页面编排，减少播放源与画质状态逻辑混杂，便于后续继续拆分事件绑定层。
- 重构后重新构建验证通过，无损优先与 HLS 按需加载行为保持不变。

## v1.0.23 (2026-03-27)

- 保持无损优先与 HLS 按需加载链路不变，为后续继续拆分播放源管理和事件绑定打基础。
- 重构后重新构建验证通过，播放器入口脚本进一步收敛。

## v1.0.22 (2026-03-27)
