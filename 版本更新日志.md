# 版本更新日志

## v1.2.1 (2026-04-02)
### 功能升级
- **全自动 VIP 转码加速**：浏览未转码视频时，后台静默模拟真实请求推入 VIP 加速转码队列。
- **批量无感排队**：只需滚动文件列表，即可自动识别页面上的未转码视频并批量加速。
- **直观状态反馈**：缩略图区域实时显示排队结果（成功排队或被拒绝原因）。

## v1.1.8 (2026-04-01)
### 功能升级
- **全新自定义移动弹窗**：彻底脱离 115 官方 JS SDK 依赖，用纯前端 + API 实现完整的移动对话框，支持文件夹浏览、新建文件夹、搜索文件夹、最近移动记录。
- **灯箱关闭优化**：点击大图或图片以外的背景区域均可关闭灯箱。

## v1.1.7 (2026-04-01)
### 问题修复
- **修复进度条预览图被视频遮挡**：将预览缩略图的 z-index 从 5 提升到 70，使其正确显示在视频层和底部控制栏之上，不再被视频画面遮挡。

## v1.1.5 (2026-03-29)
### 问题修复
- **彻底解决画质记忆失效**：重写画质匹配算法，基于标签文本（Label）而非动态 URL 进行还原，确保“115原画”在刷新后 100% 恢复。
- **解决 UI 层级遮挡**：下调进度条预览图层级（z-index: 5）并禁用交互，解决预览图覆盖画质选择菜单的问题。
- **防止记忆误覆盖**：引入 `loadedmetadata` 状态保护，避免播放器初始加载流程干扰用户持久化的画质偏好。

## v1.1.4 (2026-03-28)
- 侧栏宽度调整为 360px，文件名支持两行显示。

## v1.0.31 (2026-03-27)

## v1.0.21–v1.0.25 (2026-03-26~27)
- 项目更名为 115m，独立架构重建。
- 聚焦两项核心能力：列表预览图 + 点击无损播放。
- 播放源链路简化为 Pro API 直请求 → Web API 降级。
- Background 从 473 行精简到约 110 行。
- 播放器架构拆分：事件绑定、状态管理、HLS 加载独立模块化。
- HLS.js 改为按需动态加载，不打进首屏主包。

## v1.0.14–v1.0.20 (2026-03-25)

- 播放策略确立：无损优先直开，115 原画作为备选按需加载。
- 封面缓存增加 `chrome.storage.local` 可用性检测，不可用时降级内存缓存。
