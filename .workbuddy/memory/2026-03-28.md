# 2026-03-28

## 播放器交互修复

### 无损播放失败降级 (v1.0.33)
- **问题**: 部分视频无损播放报错 `NotSupportedError`，不会自动降级到 HLS/115原画
- **根因**: 初始化时只获取无损源或 HLS 源之一，当无损源返回 URL 但实际无效时，`m3u8List` 为空导致无法降级
- **修复**: `player.ts` 的 `init()` 改用 `Promise.all` 并行获取无损源和 HLS 源，确保 `m3u8List` 始终可用
- **日志**: `fallbackToHls()` 增加详细日志便于排查

### 播放器控件点击被拦截 (v1.0.33)
- **问题**: 画质选择、设置面板（播放速度、画面比例、画面旋转）点击均触发播放/暂停，实际功能不生效
- **根因**: `isInteractiveTarget` 漏掉了 Artplayer 的设置面板和画质面板
- **修复**: 在 `events.ts` 中增加对 `.art-settings`, `.art-setting-item`, `.art-quality`, `.art-quality-item` 的识别

### 右键菜单交互 (v1.0.33)
- **问题**: 右键菜单显示时，点击左键没有关闭菜单
- **修复**: `handleRootClick` 检测右键菜单打开状态时，先关闭菜单并阻止事件传播

### 移动后面包屑不更新 + 切换115原画失败 (v1.1.3)
- **问题1**: 文件移动成功后，左上角面包屑导航没有实时更新
- **根因1**: `Core.TreeDG.Show` 的 callback 在 115 标签页主世界执行，无法直接调用 `chrome.runtime.sendMessage`
- **修复1**: 使用自定义 DOM 事件 `115m-move-success` 从主世界传递到 content script，再由 content script 发送 `MOVE_SUCCESS_REFRESH` 消息给 background，background 通过 `chrome.tabs.sendMessage` 通知播放器页面
- **根因2**: `files/video` API 参数名错误（`pick_code` 应为 `pickcode`），且响应数据结构错误（`parent_id` 在顶层而非 `data` 内）
- **修复2**: 修正参数名为 `pickcode`，修正响应解析逻辑为 `videoResult.parent_id`
- **问题2**: 切换到"115原画"时提示"播放失败，无可用的视频源"
- **根因**: Service Worker 中动态导入 `../lib` 模块时出错（可能是 CRXJS 打包问题）
- **修复**: `background/index.ts` 的 `FETCH_M3U8` handler 改为直接使用 `fetch` API 调用 115 接口，绕过动态导入

### 画质选择按视频记忆（进行中）
- **目标**: 默认仍为"无损"，但当用户手动为某个视频切换到"115原画"或其他 HLS 画质后，只记住该视频的选择，不影响别的视频默认策略
- **第一次方案失败**: 通过 background 的 `SET_QUALITY_PREF / GET_QUALITY_PREF` + `chrome.runtime.sendMessage` 存取画质偏好，受 Service Worker 冷启动/返回 null 影响，记忆不稳定
- **第二次方案失败**: 改为在播放器页直接使用 `localStorage` 保存 `pickCode -> { label, quality }`，保存后刷新仍不生效
- **待排查原因**: `document.write` 覆盖页面可能清除了 localStorage；content script 可能运行在隔离 world
- **已记录到**: `待修复问题.md`

### 文件列表页新增操作按钮
- **新文件**: `src/content/core/action-buttons.ts`（`injectActionButtons` 函数）
- **集成**: `home.ts` 的 `scanAndRender` 中对每个视频文件注入按钮到 `.file-opr` 区域
- **115原画播放按钮**: 绿色文字，点击后打开播放器页但带 `forceQuality=hls_original` 参数（播放器页需识别该参数跳过无损获取）
- **PotPlayer 播放按钮**: 蓝色文字，通过 `FETCH_DOWNLOAD_URL` background handler 获取无损下载 URL，用 `potplayer://` URL 协议拉起
- **background handler**: `FETCH_DOWNLOAD_URL` 使用 Pro API（`/app/chrome/downurl`）获取下载地址，通过 `chrome.scripting.executeScript` 在 115 页面主世界执行 fetch

### PotPlayer 按钮点击报 "document is not defined" (v1.1.3)
- **根因**: `FETCH_DOWNLOAD_URL` handler 在 background Service Worker 中调用 `crypto115.m115_decode()`，Service Worker 没有 `document` 对象，导致报错
- **修复**: 把解密操作移到 content script 中执行。background 只返回 `{ encryptedData, key }`，content script 在有 DOM 环境的上下文中调用 `crypto115.m115_decode()` 解密
- **修改文件**: `src/background/index.ts`、`src/content/core/action-buttons.ts`、`src/shared/messages.ts`
